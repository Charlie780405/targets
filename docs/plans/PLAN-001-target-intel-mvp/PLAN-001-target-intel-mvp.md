# PLAN-001: 靶点情报引擎 MVP（纲领）

## 目标

用 6 周（6 个子计划）交付一个可独立运行的**靶点情报引擎 MVP**：聚焦 **1 个靶点 + 2–3 个自免适应症**，接入 ClinicalTrials.gov 与 PubMed，加公司/投融资，产出**人工审核后的中文周报**，并通过 Obsidian 研判 + Git 同步在混合架构（云端 Vultr 采集 / 本地研判）下持续运行。

**首个里程碑（成功定义）**：连续 4 周产出周报，重要事件漏报率可接受、重复率低、每条关键结论可追溯，并确实节省医学情报整理时间。**不是**「抓了一万条」。

## 背景与关联

- 这是「医学研发工作台」的先导独立模块；先跑通情报引擎，再接网站/浏览器扩展/更完整平台。
- 产品单位是**事件**，不是文章。详见 `docs/product-spec.md`、`docs/event-schema.md`。
- 交付流程见 `docs/agent-context/workflow.md`；工程红线见 `AGENTS.md`。

## Out of Scope（MVP 明确不做）

- 全互联网通用爬虫、所有自免靶点全覆盖
- 自动付费墙全文抓取
- 复杂知识图谱前端、自建大模型
- 一开始就开发 Obsidian 插件
- 无人工审核的自动医学结论
- 依赖搜索结果摘要作为证据
- 监管源（FDA/EMA/NMPA）、大会全自动化（MVP 只做 1 个重点大会，后续扩展）
- 向量检索 / pgvector（二阶段）

## 回滚预案

- 代码：`git revert` 对应 merge commit；子计划相互独立，可单独回滚。
- 数据：Alembic `downgrade`；原始快照保留在 `data/raw/`，可重放。
- 部署：Vultr `docker compose` 回滚到上一镜像 tag。
